# Короткий звіт до лабораторної роботи

## Тема
Розгортання Docker-проєкту в Microsoft Azure лише засобами Azure: Terraform + Cloud Shell + cloud-init.

## Мета роботи
Мета лабораторної роботи — навчитися створювати хмарну інфраструктуру в Azure за допомогою Terraform, використовувати Azure Cloud Shell як середовище керування, автоматизувати налаштування Linux VM через cloud-init і розгортати контейнеризований застосунок без використання Ansible.

## Що було реалізовано
У межах лабораторної роботи було реалізовано повністю хмарне розгортання застосунку в Microsoft Azure. Усі команди виконувалися в Azure Cloud Shell. Інфраструктура описана у вигляді Terraform-конфігурації.

Terraform створює такі ресурси:
- Resource Group;
- Virtual Network;
- Subnet;
- Public IP;
- Network Security Group;
- Network Interface;
- Linux Virtual Machine.

## Що робить cloud-init
Під час створення Linux VM через параметр `custom_data` передається сценарій `cloud-init`, який виконується при першому запуску машини. Цей сценарій:
- оновлює пакети операційної системи;
- встановлює Docker;
- встановлює Docker Compose plugin;
- клонує GitHub-репозиторій із проєктом;
- переходить у каталог із `docker-compose.yml`;
- запускає застосунок командою `docker compose up -d`.

## Як запускається Docker-проєкт
Усі компоненти системи запускаються на одній Linux VM у контейнерах Docker через Docker Compose. У складі розгортання були забезпечені:
- база даних;
- щонайменше один модуль обробки даних;
- веб-інтерфейс.

Після запуску застосунок стає доступним за публічною IP-адресою віртуальної машини.

## Як перевірялася працездатність
Працездатність перевірялася такими способами:
1. Перевірка успішного створення ресурсів через `terraform apply`.
2. Отримання `public_ip` і `app_url` через `terraform output`.
3. Перевірка HTTP-відповіді за допомогою `curl`.
4. Відкриття веб-інтерфейсу в браузері.
5. За потреби — перевірка контейнерів через SSH і команду `docker ps`.

## Висновок
У результаті лабораторної роботи було створено Terraform-конфігурацію для Azure, підготовлено cloud-init сценарій для автоматичного налаштування Linux VM, виконано автоматичне розгортання Docker-проєкту та забезпечено доступ до веб-інтерфейсу через public IP. Такий підхід дозволяє реалізувати повністю хмарний сценарій розгортання без використання локального середовища адміністрування.
